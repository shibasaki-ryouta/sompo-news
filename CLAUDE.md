# 損保業界ニュースレポート自動化タスク

このリポジトリは、週次(毎週金曜11:00 JST、手動実行も可)で国内損害保険業界のニュースレポートを作成し、
Gmail API経由でメール配信するタスクで使用する。

## メール本文フォーマット(重要な修正事項)

メール本文を組み立てる際、`reports/{today}.md` の内容(report_text)は **1行目がすでに
`# 損害保険ニュースレポート {today}` という見出しになっている**。そのため、本文の先頭に別途
「損害保険ニュースレポート {today}」という行を挿入すると、タイトルが2回表示されてしまう。

正しい本文構成:

```
※DD2部メンバーをBCCに指定しています

{report_text}
```

(誤り: 上記の間に追加で「損害保険ニュースレポート {today}」という行を挟むとタイトルが重複する)

## Gmail API 認証について

- OAuthクライアントは2026年7月に「本番環境(In production)」へ切り替え済み。これによりrefresh_tokenの
  7日失効制限は解除されている(切り替え前に発行されたrefresh_tokenは7日で失効するので使わないこと)。
- 初回同意時に「Google で確認されていないアプリです」という警告が出ることがあるが、`gmail.send`は
  Sensitive scope(Restrictedではない)のため、審査なしで「詳細」→「安全でないページに移動」で進めば
  正常に認証・送信できる。
